## 组件级完整性检查（逐组件验证，不是页面级）

> 页面级"五态 5/5"不够。**每个数据展示组件**必须独立通过以下检查：

### 列表/Feed 组件
- [ ] 有空状态（插画 + 引导文案）
- [ ] 有骨架屏加载（结构匹配真实项）
- [ ] 有错误态（重试按钮）
- [ ] 超多数据有分页/"查看更多"/虚拟滚动
- [ ] Feed 类组件有时间分组 + 新条目动画
- [ ] 图标色按类型区分（不是全部同色）

### 表格组件
- [ ] 有空状态（插画 + 引导）
- [ ] 有骨架屏加载
- [ ] >10 行有分页
- [ ] 列名超长截断
- [ ] 数字右对齐、文字左对齐

### 卡片网格组件
- [ ] 有空状态
- [ ] 有骨架屏加载
- [ ] 响应式：桌面 3-4 列 → 平板 2 列 → 手机 1 列
- [ ] 卡片内容超长截断（line-clamp）

### 图表组件
- [ ] 有无数据状态（"暂无数据" + 占位图）
- [ ] 有加载态（骨架屏或 Spinner）
- [ ] 图例/轴标签/单位清晰
- [ ] 响应式尺寸适配

## Critique Theater（自审循环，最多 3 轮）

```
第 1 轮：生成代码
    ↓
脚本扫描：运行 node scripts/quality-check.mjs <output-dir> --output output/quality-report.md
    ↓
    ↓ 发现问题？→ 修复并重新扫描
    ↓
第 2 轮：自审——对照量化清单逐项检查，发现问题立即修复
    ↓
第 3 轮：再审视——聚焦 P0 Anti-AI-Slop 和视觉节奏
    ↓
通过 → 交付。3 轮仍有问题 → 标记已知局限，交付并说明
```

**脚本与人工互补关系：**
- 脚本负责「可自动化的客观检查」：色值、字号、div onClick、占位内容
- 人工负责「需要判断力的主观检查」：对比度、五态覆盖、动效、视觉效果
- 必须先跑脚本再人工；脚本发现的违规应在修复后重新扫描确认
