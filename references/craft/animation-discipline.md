# 动效纪律 — 基于研究的动画规范

> 动效是功能，不是装饰。每一帧都有目的。

## 缓动曲线

| 场景 | 曲线 | CSS |
|------|------|-----|
| 元素进入屏幕 | 减速（Decelerate） | `cubic-bezier(0, 0, 0.2, 1)` |
| 元素离开屏幕 | 加速（Accelerate） | `cubic-bezier(0.4, 0, 1, 1)` |
| 元素在屏幕内移动 | 标准（Standard） | `cubic-bezier(0.4, 0, 0.2, 1)` |
| 弹性反馈（按钮按压） | 弹性（Spring-like） | `cubic-bezier(0.34, 1.56, 0.64, 1)` |

**`linear` 禁止用于所有过渡/动画**（进度条除外）。线性运动看起来机械、不自然。

## 时长阈值

| 尺度 | 时长 | 适用场景 |
|------|------|----------|
| 微交互 | 100-150ms（不超过 200ms） | 悬停效果、复选框切换、图标状态变化 |
| 中等过渡 | 200-300ms（不超过 400ms） | 下拉展开、卡片翻转、模态框出现 |
| 页面/区域过渡 | 300-400ms（不超过 500ms） | 页面切换、全屏遮罩、侧边栏展开 |
| 复杂编排 | 400-500ms（不超过 600ms） | 列表重排、仪表盘重新布局 |

**所有时长必须是 50 的倍数**（100/150/200/250/300/400/500ms）。

## 性能规则

```
只动画 transform 和 opacity → GPU 加速的合成层，不触发重排。
绝对不要动画 width / height / top / left / margin / padding → 触发 reflow，性能灾难。

必须维持 60fps（16.67ms 每帧预算）。
图片/视频必须预设宽高比 → 防止 CLS（Cumulative Layout Shift）。
列表动画用 will-change 或 contain 优化。
```

## 无障碍动效

```
约 35% 的成年人有前庭敏感性（运动眩晕）。
必须提供 prefers-reduced-motion 媒体查询回退：

@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}

回退策略：用淡入淡出（fade）替代位移/缩放动画，或直接瞬切。
```

## Disney 12 原则 → UI 映射

| Disney 原则 | UI 应用 |
|-------------|---------|
| **缓入缓出（Slow In/Out）** | 所有滚动、页面过渡、元素移动使用缓动曲线，永不线性。让 UI 有"重量感"而非"塑料感" |
| **挤压与拉伸（Squash & Stretch）** | 按钮按下：轻微压缩 + 回弹模拟弹性反馈。仅 100-200ms 但显著提升"触感" |
| **预备动作（Anticipation）** | 主要动作前的小反向准备动作。按钮悬停：轻微上移暗示"我可以点击" |
| **跟随与重叠（Follow Through）** | 元素停止时，相关部分（子菜单、下拉箭头）跟随稍有延迟 → 视觉层级 |
| **舞台呈现（Staging）** | 将注意力导向正确元素。过渡动画强调关键变化，弱化次要变化 |

## Material Design 3 动效系统

**过渡类型：**
- **Container Transform**：同一元素在状态间变形（列表项 → 详情页）
- **Shared Axis**：空间相关的元素一起移动（Tab 滑动）
- **Fade Through**：无关元素的交叉淡入淡出（无空间关系）

## 动效的三个功能

1. **引导注意力** — 告诉用户看哪里（新元素从哪出现）
2. **反馈状态变化** — 成功/失败的视觉确认
3. **建立空间关系** — 新页面从哪来，返回往哪去
