# Media & Video Pages — 媒体/视频类页面专项设计指南

> 视频/音频/直播页面的交互模型与文档页面完全不同——播放控制、加载体验、全屏切换和实时更新都需要专用规则。
> 从 Phase 4 布局决策树路由到本项目时加载此文件。

---

## 1. 视频播放器核心组件

### 播放器结构
```
┌──────────────────────────────────────┐
│                                      │
│            Video Player              │
│           (16:9 / 4:3 / 1:1)        │
│                                      │
│  ┌────────────────────────────────┐  │
│  │ 播放/暂停   进度条  时间 全屏  │  │
│  └────────────────────────────────┘  │
└──────────────────────────────────────┘
```

### 播放器控制栏（必须实现）

| 控件 | 要求 | 触控目标 |
|------|------|----------|
| 播放/暂停 | 大按钮，点击切换 | ≥ 44×44px |
| 进度条 | 可拖动，实时显示当前时间/总时长 | 滑块手柄 ≥ 20px |
| 音量控制 | 滑块或 +/- 按钮 | ≥ 44×44px |
| 全屏切换 | 进入/退出全屏图标 | ≥ 44×44px |
| 画中画 (PiP) | 浏览器 API 支持（Chrome/Safari/Firefox） | ≥ 44×44px |
| 字幕/CC | 切换字幕轨道 | ≥ 44×44px |
| 快进/快退 | 10s/30s 按钮（可选） | ≥ 44×44px |

### 播放器五态

| 状态 | 表现 |
|------|------|
| **Loading** | 骨架屏（16:9 灰色占位矩形 + 脉冲动画）+ 中心 Spinner |
| **Empty** | 无视频源时显示"暂无视频" + 插画 + 上传/选择视频按钮 |
| **Error** | 播放失败提示 + 错误原因 + 重试按钮（非自动重试） |
| **Populated** | 正常播放（控制栏 2s 无操作自动隐藏，鼠标移动恢复） |
| **Edge** | 网络波动提示（"缓冲中"覆盖层）、分辨率自适应降级 |

## 2. 播放列表组件

### 列表结构
```
┌──────────────────────────────┐
│ 播放列表标题               │
│                              │
│ ┌──── ────────────────────┐ │
│ │缩略图 │ 标题            │ │
│ │       │ 作者 · 时长     │ │
│ ├──── ────────────────────┤ │
│ │缩略图 │ 标题（当前播放）│ │
│ │       │ 作者 · 时长     │ │
│ ├──── ────────────────────┤ │
│ │缩略图 │ 标题            │ │
│ │       │ 作者 · 时长     │ │
│ └──── ────────────────────┘ │
│                              │
│ [查看更多 / 加载更多]        │
└──────────────────────────────┘
```

### 播放列表五态

| 状态 | 表现 |
|------|------|
| **Loading** | 3-5 个骨架屏项（灰色矩形缩略图 + 2 行文字占位） |
| **Empty** | 插画 + "暂无视频" + "去上传视频"按钮 |
| **Error** | "加载失败" + 重试按钮 |
| **Populated** | 列表正常显示，**当前播放项高亮**（背景+左侧色条） |
| **Edge** | >50 项时分页或虚拟滚动；标题超长 ellipsis；时长显示 "99:59:59" 格式 |

### 列表项规范
- 缩略图宽高比：**16:9**（必须预设宽高比防止 CLS）
- 缩略图尺寸：160×90px（紧凑） / 320×180px（标准）
- 标题：单行 ellipsis，≤ 2 行
- 时长：右上角标注 `MM:SS` 或 `HH:MM:SS`
- 当前播放项：左侧 3px 主色条 + 浅色背景

## 3. 缩略图规范

### 占位图规则
```
开发阶段：https://placehold.co/320x180/e2e8f0/64748b?text=视频封面
生产阶段：使用真实视频截图或上传封面

必须预设宽高比，防止 CLS：
<img src="..." width="320" height="180" style="aspect-ratio: 16/9" loading="lazy">
```

### 缩略图状态
| 状态 | 表现 |
|------|------|
| 正常 | 16:9 封面图，加载后渐入（opacity 0 → 1, 300ms） |
| 加载中 | 灰色占位 + 脉冲动画（同骨架屏） |
| 加载失败 | 16:9 灰底 + 播放图标 + "封面加载失败" |
| 无封面 | 16:9 品牌色底 + 视频类型图标 |

## 4. 自动播放规则

```
绝对规则：
❌ 禁止自动播放有声视频（违反 Chrome/Safari 自动播放策略）
❌ 禁止页面加载即播放（用户需主动触发）

✅ 静音自动播放：允许（如 Hero 背景视频），但需有"解除静音"按钮
✅ 用户操作后自动播放：用户点击播放后，下一个视频可自动连播
✅ 滚动进入视口自动播放：仅限静音，且 Intersection Observer 检测 >50% 可见
```

### 自动播放检测
```javascript
// 浏览器自动播放策略检测
const video = document.querySelector('video');
video.play().catch(() => {
  // 自动播放被阻止，显示播放按钮覆盖层
  showPlayOverlay();
});
```

## 5. 字幕与可访问性

### 字幕要求
- 视频必须有字幕（至少提供 CC 轨道）
- 字幕轨道使用 `<track>` 标签（WebVTT 格式）
- 字幕字体大小：相对于视频高度的 **4-6%**
- 字幕背景：半透明黑底，白色文字，保证对比度 ≥ 4.5:1
- 字幕区域不遮挡视频关键内容（下端 1/3 区域）

```html
<video controls>
  <source src="video.mp4" type="video/mp4">
  <track kind="subtitles" src="subtitles.vtt" srclang="zh" label="中文">
  <track kind="subtitles" src="subtitles-en.vtt" srclang="en" label="English">
</video>
```

### 音频替代
- 纯音频内容必须有文字转录（Transcript）
- 视频的音轨关键信息（对话、旁白）需在页面提供文字摘要

## 6. 全屏与画中画

### 全屏 API
```javascript
// 进入全屏
element.requestFullscreen();

// 退出全屏
document.exitFullscreen();

// 全屏变化事件
document.addEventListener('fullscreenchange', () => {
  updateUIForFullscreen(!!document.fullscreenElement);
});
```

### 全屏模式规则
- 全屏按钮在控制栏恒常可见
- 全屏时控制栏自动隐藏（2s 无操作），任意触摸显示
- 全屏退出按钮：Esc 键 + 控制栏退出按钮
- 全屏时锁定屏幕方向为横屏（建议）

### 画中画 (Picture-in-Picture)
- 画中画按钮在控制栏可选
- PiP 窗口尺寸：最小 320×180px，不超过屏幕 30%
- PiP 窗口包含：视频画面 + 播放/暂停 + 关闭按钮

## 7. 直播场景专项

### 直播 UI 组件
```
┌──────────────────────────────────────┐
│                      ● LIVE 12,345   │
│                                      │
│            Video Player              │
│                                      │
│  ┌────────────────────────────────┐  │
│  │ 聊天面板（实时滚动）           │  │
│  │ 用户1: 来了来了               │  │
│  │ 用户2: 主播加油👏            │  │
│  │ [输入框...] [发送]            │  │
│  └────────────────────────────────┘  │
└──────────────────────────────────────┘
```

### 直播特有状态
| 状态 | 表现 |
|------|------|
| **直播中** | 红色 LIVE 徽章 + 脉冲动画 + 观看人数 |
| **直播结束** | "直播已结束" 覆盖层 + 回放按钮（如有） |
| **等待开始** | 倒计时 / "直播将在 XX:XX 开始" + 预约按钮 |
| **缓冲中** | "网络不稳定" 提示 + 旋转 Spinner（不覆盖整个画面） |
| **断流** | "主播暂时离开" 占位图 + 自动重连指示器 |

### 直播聊天规范
- 实时滚动：新消息从底部滑入，不打断当前阅读位置
- 消息上限：最多保留 200 条，超出后丢弃最旧消息
- 频率控制：500ms 内最多显示 5 条消息，防刷屏
- 输入框始终可见，发送按钮 ≥ 44×44px

## 8. 媒体控制器触控目标

```
所有交互控件触控目标 ≥ 44×44px（Apple HIG）：
┌─────────────┬──────────┬──────────────┐
│ 播放/暂停   │ 48×48px  │ 核心操作     │
│ 快进/快退   │ 44×44px  │ 快速定位     │
│ 音量 +/-    │ 44×44px  │ 辅助控制     │
│ 全屏切换    │ 44×44px  │ 模式切换     │
│ 画中画      │ 44×44px  │ 模式切换     │
│ 字幕切换    │ 44×44px  │ 辅助控制     │
│ 进度条拖拽  │ 滑块20px │ 整个条可触   │
└─────────────┴──────────┴──────────────┘

控制栏高度：桌面 40-48px，移动端 48-56px（含安全区）
控制栏间距：控件之间至少 12px（防误触）
```

## 9. 五态覆盖检查清单

媒体页面交付前确认：

- [ ] 播放器：有 Loading（骨架屏）/ Empty / Error / Populated / Edge 覆盖
- [ ] 播放列表：有 Loading / Empty / Error / Populated / Edge 覆盖
- [ ] 缩略图：正常/加载中/加载失败/无封面 四种状态
- [ ] 自动播放：禁止有声自动播放，静音自动播放有"解除静音"按钮
- [ ] 字幕：使用 `<track>` 标签，对比度 ≥ 4.5:1
- [ ] 全屏：Esc 退出 + 控制栏按钮 + 方向锁定（建议）
- [ ] 直播：LIVE 徽章 + 直播结束/等待/断流状态覆盖
- [ ] 触控：所有控件 ≥ 44×44px
- [ ] 性能：图片/视频预设宽高比防止 CLS
- [ ] 响应式：控制栏在 375px 不溢出，文字不重叠

## 铁律

```
铁律: 禁止有声自动播放。违反浏览器策略，导致用户体验严重下降。
铁律: 每个媒体控件触控目标 ≥ 44×44px。视觉可以小，触控区域必须大。
铁律: 所有缩略图和视频容器必须预设宽高比（aspect-ratio 或 width/height），防止 CLS。
铁律: 直播场景必须有 LIVE 徽章 + 直播结束/断流/等待状态覆盖。
```
