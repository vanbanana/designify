# Design Character — 设计性格系统

> 设计不是"好看"，设计是"有性格"。
> 一个没有性格的页面 = 一个没有人记住的页面。

---

## 核心概念

每个页面完成后必须能回答：**"关掉这个页面后，用户能记住的一个东西是什么？"**

这个问题的答案就是页面的设计性格（Design Character）。它是一个可以通过三个独立维度来量化的属性：

```
大胆度（Boldness）  — 页面敢于打破常规的程度
动效度（Motion）    — 页面自己会"动"的程度
密度（Density）    — 页面塞满信息的程度
```

**这三个维度不是审美偏好，它们对应三组认知心理学原理，每个在 designify 的 laws-of-ux.md 中都有明确的依据。**

---

## 三轴定义

### 轴 1：大胆度（Boldness）1-10

| 值 | 描述 | 视觉表现 |
|----|------|----------|
| 1-3 | **保守安全** — 符合所有约定，无意外 | Ant Design 默认、标准表单布局、传统卡片网格 |
| 4-6 | **适度个性** — 有一个令人注意的差异 | 一个不对称布局、一个大胆的字体选择、一个意外的留白 |
| 7-8 | **明确态度** — 设计有自己的声音 | 大字号对比（4x+）、色块分割而非线框、独特的视觉叙事 |
| 9-10 | **实验突破** — 设计本身是体验的一部分 | 非常规布局、动态排版、交互式背景、艺术性纹理 |

**心理学依据：**
- **Von Restorff 效应**（laws-of-ux.md）：与众不同的元素更容易被记住。Boldness 越高，页面被记住的概率越大。
- **Aesthetic-Usability**（laws-of-ux.md）：好看的设计被感知为更好用。但 Boldness ≠ 好看，Boldness = 有意图的与众不同。
- **Jakob's Law**（laws-of-ux.md）：用户大部分时间花在别人的产品上。Boldness > 7 时必须在细节上创新，不能在核心可用性上创新。

### 轴 2：动效度（Motion）1-10

| 值 | 描述 | 视觉表现 |
|----|------|----------|
| 1-2 | **纯静** — 无任何动效 | 后台管理页面、数据表格 |
| 3-4 | **克制** — 仅有状态反馈动效 | 按钮 hover、焦点过渡、简单的骨架屏 |
| 5-6 | **适度** — 有引导性动效 | 页面过渡、滚动淡入、导航切换缓动 |
| 7-8 | **丰富** — 动效是体验的一部分 | 滚动视差、入场序列、微交互反馈 |
| 9-10 | **叙事** — 动效是主要内容载体 | 滚动故事线、全屏过渡、标记性动画 |

**心理学依据：**
- **Doherty Threshold**（laws-of-ux.md）：响应超过 400ms 打断注意力。Motion 不是装饰，是缩短感知时间的手段。
- **Zeigarnik Effect**（laws-of-ux.md）：未完成的任务更容易被记住。序列动画利用了这种"未完待续"感。
- **Peak-End Rule**（laws-of-ux.md）：用户只记得最高点和结尾。关键动效应该出现在最高点或结尾。

### 轴 3：信息密度（Density）1-10

| 值 | 描述 | 视觉表现 |
|----|------|----------|
| 1-2 | **画廊式** — 极多留白，一屏显示极少信息 | 品牌首页、艺术作品展示、Hero 页面 |
| 3-4 | **宽松** — 内容呼吸，有充足留白 | 营销落地页、文章页面、个人作品集 |
| 5-6 | **均衡** — 内容与留白平衡 | SaaS 主页、电商分类页、博客列表 |
| 7-8 | **密集** — 信息优先，留白克制 | 仪表盘、数据分析页、项目管理 |
| 9-10 | **极密** — 每一像素都为信息服务 | 交易终端、监控面板、编辑器界面 |

**心理学依据：**
- **Hick's Law**（laws-of-ux.md）：选项越多，决策时间越长。Density > 7 时必须用渐进式披露补偿。
- **Miller's Law (7±2)**（laws-of-ux.md）：短期记忆容量有限。Density > 7 时必须分块、视觉层级不超过 4 层。
- **Gestalt: Proximity**（laws-of-ux.md）：靠近的元素被感知为一组。Density > 6 时分组必须清晰，否则用户无法快速扫描。

## 如何用在 Design Read 中

在 Phase 3（Design Read）中，必须输出设计性格声明：

```
## Design Character

**大胆度（Boldness）：** 6/10（适度的个性——在 hero 区域使用不对称分割，其他区域保持标准）
  → 心理学依据：Von Restorff 效应——不对称分割是用户记住的第一个东西

**动效度（Motion）：** 4/10（克制——仅 scroll reveal + 按钮反馈，无叙事动效）
  → 心理学依据：Doherty Threshold ——低于 400ms 的过渡不会打断用户心流

**密度（Density）：** 4/10（宽松——每屏聚焦一个核心信息，大留白）
  → 心理学依据：Hick's Law ——每屏不超过 7 个选项

**令人难忘的品质：** "Hero 区域的不对称分割——左侧大标题+右侧全高配色块"
  → 验证：关掉页面后，用户会记得"那个不对称的首页"
```

---

## 铁律

```
铁律: 每个页面必须有设计性格声明（Boldness / Motion / Density）。
铁律: Design Read 中必须输出三轴值+心理学依据。
铁律: Phase 6 质量门禁必须验证最终输出与声明的设计性格一致。
铁律: "令人难忘的品质"说不出来 = 页面不合格。
```

## 红旗列表

| 你在想... | 现实 |
|-----------|------|
| "所有页面 Boldness=5 最安全" | Boldness=5 = 平均 = 没人记住。选择极端的值并证明它。 |
| "Motion 少一点总不会错" | Motion 少=Boldness 高需要的补偿。零 Motion 的页面必须用其他方式抓住注意力。 |
| "密度中等最稳妥" | 中等密度 = 中庸。要么画廊式留白（Density 2-3）要么信息丰富（Density 7-8），中间是最难出彩的。 |
| "设计性格确定后不能变" | 不同页面可以有不同性格。首页 Boldness=7，详情页 Boldness=4 完全合理。但同一页内必须一致。 |

## 合理化预防

| 借口 | 现实 |
|------|------|
| "这个页面只是简单的表单，不需要设计性格" | 表单页面也有性格：密度高（7）+ 动效低（2）+ 大胆度中（4）同样是一种性格。无声明 ≠ 无性格。 |
| "Boldness=3 意味着设计很无聊" | Boldness=3 的意思是"信任优先"。适合金融、医疗、政务。不一致才是问题，3 本身不是问题。 |
| "我记不住要输出设计性格" | Design Read 模板中有 Design Character 字段。不是备忘录，是模板的必填部分。 |
