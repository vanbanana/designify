# Spacing Craft — 间距与布局工艺

> 通用间距和布局规则，适用于任何品牌和风格。

## 间距网格系统（4px 基础单位）

**所有间距值必须是 4 的倍数。** 优先使用 8 的倍数，4px 仅用于微调：

```
4px   (1×4)   仅用于图标与文字的间距、紧凑元素内微调
8px   (2×4)   元素内间距、相邻元素间距（推荐默认）
12px  (3×4)   紧凑组件内部间距（标签组、按钮组）
16px  (4×4)   组件内间距、相关元素组
20px  (5×4)   紧凑卡片内间距
24px  (6×4)   卡片内间距、段落间距
32px  (8×4)   区域间距
48px  (12×4)  Section 间距
64px  (16×4)  大 Section 间距
96px  (24×4)  Hero 区域上下间距
128px (32×4)  页面级分隔（仅大屏）
```

**CSS 变量实现：**
```css
:root {
  --space-1: 4px;
  --space-2: 8px;
  --space-3: 16px;
  --space-4: 24px;
  --space-5: 32px;
  --space-6: 48px;
  --space-7: 64px;
  --space-8: 96px;
}
```

## 响应式断点

```css
/* 必须处理的 3 个核心断点 */
@media (max-width: 375px)  { /* 手机竖屏 */ }
@media (min-width: 768px)  { /* 平板 */ }
@media (min-width: 1440px) { /* 桌面 */ }

/* 可选扩展断点 */
@media (min-width: 1024px) { /* 小桌面/横屏平板 */ }
@media (min-width: 1920px) { /* 大屏 */ }
```

### 每个断点的布局策略

**375px（手机）：**
- 单列布局
- 导航收起为汉堡菜单
- 卡片全宽
- 侧边栏隐藏（抽屉式）
- 底部 Tab 导航代替顶部导航
- 字号：正文 14px，标题 20-24px
- 触摸目标 ≥ 44px

**768px（平板）：**
- 可以利用双列
- 侧边栏可以显示（窄版）
- 网格从 1 列变为 2 列
- 不要简单拉伸手机布局——要利用额外宽度

**1440px（桌面）：**
- 内容最大宽度 1200px（居中）或 1440px（全宽应用）
- 侧边栏正常显示
- 三列或四列网格
- 悬停交互可用（tooltip、dropdown）

## 容器最大宽度

| 内容类型 | max-width | 理由 |
|----------|-----------|------|
| 文章/阅读 | 680-720px | 每行 45-75 个中文字符最佳阅读宽度 |
| 表单 | 480-640px | 表单太宽用户视线跳跃 |
| 营销落地页 | 1200px | 需要视觉冲击力 |
| 后台应用 | 无限制（100%） | 利用全部屏幕空间 |
| 仪表盘 | 无限制（100%） | 数据密度需要空间 |

## 留白哲学

**留白是设计元素，不是浪费空间。**

- Section 之间的间距 ≥ Section 内部间距的 2 倍（亲密性原则）
- 标题和它下方内容的间距 < 标题和它上方内容的间距（标题属于下方内容）
- 相关元素的间距 < 不相关元素的间距

## 布局反模式

- ❌ 所有内容挤在一起（间距 < 8px）
- ❌ 所有间距都一样（没有层级感）
- ❌ 手机端用 hover 交互（没有 hover 状态）
- ❌ 桌面端内容超过 1200px 不限制宽度（阅读体验差）
- ❌ 响应式只是缩小字号（应该重新排列布局）

## 嵌套递进规则

```
父容器 padding > 子元素 gap > 元素间距。逐级递减 4 或 8px。

示例：
页面边距 24px → 卡片间距 16px → 卡片内部元素间距 8px
侧边栏 padding 24px → 菜单组间距 16px → 菜单项间距 4px
模态框 padding 32px → 区块间距 24px → 表单项间距 16px
```

## 组件专用间距

| 场景 | 推荐间距 | 说明 |
|------|----------|------|
| 按钮水平内边距 | 16-24px | 水平 = 垂直的 2-3 倍 |
| 卡片内边距 | 16-24px | 密集=16，宽松=24 |
| 模态框内边距 | 24-32px | 大容器用大间距 |
| 组件之间 | 16-32px | 相关=16，无关=32 |
| 区块之间 | 32-64px | 区块过渡用大间距 |
| 图标与文字 | 8px | 不超过 12px，否则视觉断裂 |
| Grid/Flex gap（紧凑） | 8px | 标签组、按钮组 |
| Grid/Flex gap（默认） | 16px | 卡片列表、表单项 |
| Grid/Flex gap（宽松） | 24px | 仪表盘模块 |

## 铁律

```
铁律: 所有间距值必须是 4 的倍数。没有例外。
铁律: 嵌套递进必须递减。父 > 子 > 孙，每级减 4-8px。
```

## 红旗列表

| 你在想... | 现实 |
|-----------|------|
| "10px 看起来更合适" | 10 不是 4 的倍数。用 8px 或 12px。 |
| "这个间距差不多" | "差不多"不是精确值。查数值。 |
| "设计师用的 5px" | 设计师也可能犯错。4 的倍数是系统一致性要求。 |
| "嵌套不需要严格递减" | 不递减 = 视觉层级混乱。用户感知得到但说不出来。 |

## 合理化预防

| 借口 | 现实 |
|------|------|
| "这个特殊场景需要非标准值" | 不存在特殊场景。4 的倍数覆盖从 4px 到 128px 的全部需求。 |
| "差 1px 看不出来" | 1px 偏差在 Retina 屏上变成 2px。累积起来就是视觉混乱。 |
| "我用了 rem 就不用管了" | rem 也需要是 4 的倍数（0.25rem = 4px, 0.5rem = 8px）。 |
