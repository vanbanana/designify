# Anti-AI-Slop — 反 AI 视觉臭味规则

> AI 生成的前端代码有明显的视觉"臭味"——一眼就能看出是 AI 做的。以下规则帮你避免这些特征。

### 严重程度分级

| 级别 | 含义 | 处理方式 |
|------|------|----------|
| **P0（必须修复）** | 一眼就能看出是 AI 生成的致命特征 | 生成代码时自动检查，出现即返工 |
| **P1（应该修复）** | 降低设计质量的明显问题 | 人工审查时必须修改 |
| **P2（建议修复）** | 区分"合格"和"出色"的细节 | 有时间就打磨 |

## 颜色臭味

| 严重程度 | 臭味特征 | 为什么是臭味 | 修正方案 |
|----------|----------|-------------|----------|
| P0 | `#6366f1` 做主色 | Indigo-500 是多数 LLM 的默认主色 | 选有辨识度的品牌色 |
| P0 | 紫蓝渐变背景 | AI 最爱的"科技感"捷径 | 用纯色或微妙渐变 |
| P0 | 彩虹色卡片边框 | 看似炫酷实则廉价 | 用统一的边框色 |
| P0 | 所有状态色同饱和度 | 视觉上不协调 | success/error/warning 统一调性 |

## 布局臭味

| 严重程度 | 臭味特征 | 为什么是臭味 | 修正方案 |
|----------|----------|-------------|----------|
| P0 | 三列等宽卡片 | 最偷懒的布局，没有视觉节奏 | 用不对称网格（2:1, 1:2, 3:2） |
| P0 | 图标+标题+描述千篇一律 | 每个模块长得一样 | 交替使用左图右文/上图下文/纯文字 |
| P1 | 所有 Section 等高 | 不自然的整齐 | 让内容决定高度 |
| P1 | Hero 总是"大标题+副标题+两个按钮" | 最通用的 AI Hero 公式 | 根据产品特性设计 Hero |

## 组件臭味

| 严重程度 | 臭味特征 | 为什么是臭味 | 修正方案 |
|----------|----------|-------------|----------|
| P0 | 所有按钮都是实心圆角+阴影 | 没有层级区分 | Primary 实心，Secondary 描边，Tertiary 文字 |
| P0 | 所有卡片 `border-radius: 12px` + `box-shadow: sm` | 千篇一律 | 用 6-8px，用边框代替阴影 |
| P1 | 模糊玻璃卡片滥用 | `backdrop-filter: blur` 到处用 | 只在浮层（dropdown/modal）使用 |
| P0 | 装饰性渐变色 blob | 不服务于功能的背景装饰 | 删除或用品牌相关的图形 |
| P1 | 所有输入框都是同样大小 | 没有根据内容调整 | 搜索框窄，描述框宽，验证码框短 |

## 内容臭味

| 严重程度 | 臭味特征 | 为什么是臭味 | 修正方案 |
|----------|----------|-------------|----------|
| P0 | "Lorem ipsum" 占位 | 看不出产品真实场景 | 用符合业务的中文内容 |
| P0 | "John Doe" / "Acme Corp" | AI 最爱的假名 | 用真实感的中文名字和公司名 |
| P1 | 功能描述全是"快速""智能""安全" | 空洞的 AI 话术 | 用具体的功能描述 |
| P0 | 所有用户评价都是5星+长段好评 | 不真实 | 有不同评分，评价简短真实 |
| P1 | 价格总是 $9.99 / $29.99 / $99.99 | AI 定价模板 | 根据实际业务定价 |

## 交互臭味

| 严重程度 | 臭味特征 | 为什么是臭味 | 修正方案 |
|----------|----------|-------------|----------|
| P0 | 只有默认状态 | AI 只写 happy path | 必须有空/加载/错误/部分加载状态 |
| P0 | 所有按钮点击无反馈 | 没有交互感 | 添加 active 缩放/颜色变化 |
| P0 | 表单无验证 | AI 不处理错误场景 | 实时验证+提交验证+错误提示 |
| P1 | 列表没有空状态 | "暂无数据"一行字 | 设计引导性的空状态（图标+说明+操作按钮） |
| P1 | 加载时全白 | 无骨架屏 | 用骨架屏模拟内容结构 |

## 核心原则

**每个页面问自己：如果用户看到这个页面，他能一句话说出这是什么产品吗？**

如果不能，说明设计太通用，需要加入产品特有的内容和结构。

## Before / After 修复示例

### 1. 颜色臭味修复

```
❌ Before (AI 味):
background: linear-gradient(135deg, #6366f1, #8b5cf6);

✅ After (有设计意图):
background: var(--color-primary);  /* 从品牌 DESIGN.md 提取的真实主色 */
```

### 2. 布局臭味修复

```
❌ Before (AI 味 — 三列等宽卡片):
display: grid; grid-template-columns: repeat(3, 1fr); gap: 24px;

✅ After (有信息层级):
display: grid; grid-template-columns: 2fr 1fr 1fr; gap: 16px;
/* 主内容占 2 份，次要内容各 1 份 */
```

### 3. 组件臭味修复

```
❌ Before (AI 味 — 所有按钮一样):
.btn { background: #6366f1; color: white; padding: 12px 24px; border-radius: 12px; }

✅ After (有按钮层级):
.btn-primary { background: var(--primary); color: #fff; padding: 10px 24px; border-radius: 8px; }
.btn-secondary { background: transparent; color: var(--primary); border: 1px solid var(--primary); padding: 10px 24px; border-radius: 8px; }
.btn-ghost { background: transparent; color: var(--text-secondary); padding: 10px 24px; }
```

## 铁律

```
铁律: P0 项出现即返工。不存在"差不多可以"的 P0。
```

## 红旗列表 — 你在合理化时的信号

| 你在想... | 现实 |
|-----------|------|
| "这个渐变其实挺好看的" | 紫蓝渐变是 AI 默认输出。"好看"不等于"有设计意图"。 |
| "用户没说不行" | 用户不知道你用了 AI 默认值。你的职责是主动消除 AI 味。 |
| "就一个地方用了 indigo" | 一个 P0 = 整体 AI 味。P0 不是计分制。 |
| "这个卡片布局是最合理的" | 三列等宽是最偷懒的，不是最合理的。考虑主次关系。 |
| "lorem ipsum 后面会替换" | 后面永远不会替换。现在就写真实内容。 |
| "空状态不重要" | 空状态是用户的第一印象。它比有数据状态更重要。 |

## 证据先于声明

```
❌ "这个设计没有 AI 味"
✅ "检查清单：P0 项 0/6 出现，P1 项 0/5 出现。无 AI 味。"

没有跑检查清单就声称没有 AI 味 = 不诚实。
```
