## 调色板结构（最小 8 色，最大 15 色）

```
语义色（必须）：
--color-primary        主色（品牌色，每屏最多出现 2-3 次）
--color-secondary      辅助色
--color-success        成功（绿）
--color-warning        警告（黄/橙）
--color-error          错误（红）
--color-info           信息（蓝）

中性色（必须）：
--color-bg             背景
--color-surface        卡片/面板
--color-text           主文字
--color-text-secondary 辅助文字
--color-text-muted     最弱文字
--color-border         边框
--color-divider        分割线
```

## 主色使用规则

**每屏主色出现次数 ≤ 3 次：**
1. 主 CTA 按钮
2. 当前激活的导航/标签
3. 关键链接或图标

**主色不要用在：**
- 大段背景色（用主色的 5-10% 透明度版本）
- 正文文字
- 边框（除非是 focus ring）
- 装饰性元素

## 对比度规则（WCAG AA）

| 元素类型 | 最低对比度 | 推荐对比度 |
|----------|-----------|-----------|
| 正文文字 | 4.5:1 | 7:1 |
| 大文字（>18px bold） | 3:1 | 4.5:1 |
| 图标/UI组件 | 3:1 | 4.5:1 |
| 焦点指示器 | 3:1 | - |
| 装饰性元素 | 无要求 | - |

**快速验证：** 浅灰背景 (#f5f5f5) 上的文字至少要是 #595959 或更深

## 暗色模式

**不是简单地反转颜色。** 暗色模式的规则：

```
背景层级（从深到浅）：
Level 0: #0a0a0a ~ #121212    页面背景
Level 1: #1a1a1a ~ #1e1e1e    卡片/面板
Level 2: #2a2a2a ~ #2e2e2e    悬浮/弹出

文字层级（从亮到暗）：
Primary:   rgba(255,255,255,0.87) ~ #e0e0e0    主文字
Secondary: rgba(255,255,255,0.60) ~ #a0a0a0    辅助
Muted:     rgba(255,255,255,0.38) ~ #666666    最弱

关键区别：
- 暗色模式不用纯白文字(#fff)，用 rgba(255,255,255,0.87) 避免刺眼
- 暗色模式不用纯黑背景(#000)，用 #0a0a0a~#121212
- 暗色模式的阴影改为更亮的边框（rgba(255,255,255,0.05~0.1)）
- 主色在暗色模式下需要降低饱和度 10-20% 才能避免视觉振动（高饱和色在深色背景上刺眼）
```

## 色彩反模式

- ❌ 用 `#6366f1`（Indigo 500）做主色 — 这是 AI 生成的默认色
- ❌ 用 `#3b82f6`（Blue 500）做主色 — 太常见，没有辨识度
- ❌ 用渐变做按钮背景 — 除非品牌明确规定
- ❌ 用彩色背景+彩色文字 — 对比度一定不够
- ❌ 用纯红(#ff0000)做错误色 — 太刺眼，用 #dc2626 或 #ef4444
- ❌ 所有状态色都用同一个饱和度 — success 绿和 error 红应该视觉权重一致

## 60-30-10 法则

```
配色比例：
- 60% 主色/背景色（通常是中性色：白、灰、米）
- 30% 辅助色（卡片、次要元素、区块背景）
- 10% 强调色（CTA 按钮、重要链接、图标高亮）

调色板最多 5 种颜色。
验证方法：将设计转为灰度，如果层级仍然清晰，说明配色合理。
```

## 强调色配给制（Accent Rationing）

> 颜色是**珠宝**，不是涂料。用得越少，越有力。

60-30-10 法则定义了颜色的面积比例，但没有定义**出现次数**。一个常见的"朴素"问题是：所有卡片都有品牌色强调、所有图标都用主色、导航高亮+CTA按钮+链接都用同一种颜色——结果就是"每个地方都有颜色，但每个地方都不突出"。

### 核心规则

**主色（Primary Color）每屏出现次数 ≤ 3 次：**

1. **主 CTA 按钮**（必有一次）
2. **当前激活的导航/标签**（可选）
3. **一个关键链接或图标**（可选，最多一个）

**为什么是 3 次？**
- **Von Restorff 效应**（laws-of-ux.md）：与众不同才能被记住。如果到处都是品牌色，品牌色就不再"与众不同"。
- 视觉对比度 = 稀有度。第 1 次出现最有冲击力，第 2 次仍然有效，第 3 次就开始稀释，第 4 次之后用户已经"习惯"了品牌色。

### 不要做的事情

```
❌ 每张卡片都用品牌色作为左边界强调色 → 品牌色被稀释，没人在意
❌ 所有图标都用品牌色 → 用户区分不了功能图标和品牌标识
❌ 主色既做按钮又做链接又做背景装饰 → 多用途 = 无重点
❌ 一个页面 4+ 个主色出现点 → 视觉疲劳，用户闭着眼睛都能猜到品牌色

✅ 一张卡片用品牌色突出（最重要的那张），其余卡片用中性色
✅ 只有最重要的那个图标用品牌色，其余用 muted 灰色
✅ 主色只出现在 CTA 按钮上，导航高亮用更克制的处理
```

### "一个大胆之举"原则

每个页面最多有 **1 个最大胆的颜色决策**。这个决策可以是：

- 一个超大字号标题使用品牌色（而非中性色）
- 一个色块分割布局（品牌色占 1/3 宽度）
- 一个完全品牌色的 Section 背景

**规则：** 如果你选择了"色块分割"作为大胆之举，那 CTA 按钮就用中性色（描边按钮而非实心按钮）。页面上只能有一个"主角"。

### 配色检查清单

生成代码后逐个确认：
- [ ] 主色在本屏出现了几次？≤ 3 次？
- [ ] 有"一个大胆之举"吗？它是唯一的还是和其他颜色元素竞争注意力的？
- [ ] 如果把主色全部去掉换成灰色，页面功能是否仍然可用？如果能，说明主色是装饰，不是必需品
- [ ] 每个使用主色的元素都问一句：非用不可吗？
- [ ] 装饰性元素（icon、侧边色条、背景块）用的主色还是中性色？优先用中性色

## 色彩心理学

| 颜色 | 心理效应 | 常见用途 |
|------|----------|----------|
| 红色 | 紧迫感、兴奋、危险 | 促销 CTA、限时抢购、错误状态 |
| 蓝色 | 信任、专业、安全 | 科技/金融/医疗品牌 |
| 绿色 | 自然、成长、成功 | 成功状态、环保品牌、健康 |
| 黄色 | 乐观、温暖（过度使用=视觉疲劳） | 警示状态、活泼品牌 |
| 紫色 | 奢华、创造力 | 美妆、高端品牌 |
| 橙色 | 活力、友好 | 次级 CTA、社交品牌 |
| 黑色 | 权威、精致 | 奢侈品牌、正文 |
| 白色 | 纯净、简洁 | 背景、留白 |

## 色彩组合理论

```
互补色（Complementary）：色轮对面（红/绿）。最大对比，适合 CTA 强调。
类似色（Analogous）：色轮相邻 2-3 色（蓝/蓝绿/绿）。和谐平静，适合大面积使用。
三色组（Triadic）：色轮等距 3 色（红/黄/蓝）。丰富但需控制比例。
分裂互补（Split-Complementary）：互补 + 其两侧邻色。保留对比，减少冲突。
```

## 文化色彩差异

```
白色：西方=纯洁/婚礼 vs 东亚=丧事/哀悼
绿色：西方=自然/环保 vs 部分东方文化=出轨/背叛
红色：西方=危险/停止 vs 中国=幸运/繁荣/春节
全球产品必须根据目标市场本地化色彩选择。
```

## 暗色模式饱和度

```
暗色模式中品牌色饱和度降低 10-20%（不是提亮，是降饱和）。
高饱和色在深色背景上会产生视觉振动（chromatic vibration），造成眼疲劳。
```

## 铁律

```
铁律: 没有 60-30-10 比例就不能确定配色方案。
铁律: 对比度不达标就不能交付。正文 ≥ 4.5:1，大文字 ≥ 3:1。
铁律: 暗色模式必须降饱和 10-20%，不是提亮。
```

## 合理化预防

| 借口 | 现实 |
|------|------|
| "这个颜色搭配看起来不错" | "看起来"不是证据。用 60-30-10 法则验证比例。 |
| "用户喜欢这个颜色" | 用户喜欢 ≠ 对比度达标。WCAG 是法律要求，不是建议。 |
| "暗色模式提亮一点更醒目" | 提亮 = 视觉振动。降饱和才是正确做法。 |
| "4.5:1 太严格了" | 4.5:1 是 WCAG AA 最低合规线。不是严格，是底线。 |
| "这个灰色足够深了" | 用对比度计算器验证。感觉 ≠ 事实。 |
