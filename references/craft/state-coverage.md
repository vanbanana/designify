# 状态覆盖 — 五态完整性矩阵

> 每个 UI 组件和数据源都必须覆盖 5 种状态。缺一种 = 不合格。

## 五态定义

| 状态 | 定义 | 常见表现 |
|------|------|----------|
| **Loading（加载中）** | 数据正在获取 | 骨架屏（结构匹配真实内容）、Spinner、进度条 |
| **Empty（空数据）** | 数据获取成功但为空 | 插画 + 文案 + 行动按钮（不是简单的"暂无数据"） |
| **Error（错误）** | 获取失败或操作失败 | 错误插画 + 原因说明 + 重试按钮 |
| **Populated（有数据）** | 正常展示 | 数据列表/卡片/图表 |
| **Edge（边界）** | 极端数据情况 | 超长文本、超多数据、极端数字、无空格字符串 |

## 各状态详细规范

### Loading 状态
```
首次加载：骨架屏（Skeleton）——结构必须匹配真实内容的布局
翻页/加载更多：局部加载指示器（不是全屏遮罩）
下拉刷新：释放触发 → 刷新动画 → 完成收起
超时（>10 秒）：显示"加载较慢" + 重试按钮
加载失败：错误插画 + 错误原因 + 重试按钮
```

### Empty 状态
```
列表为空：插画 + "暂无数据" + 行动按钮（如"创建第一个项目"）
搜索无结果："未找到相关内容" + 搜索建议（试试更短的关键词/检查拼写）
筛选无结果："没有匹配项" + 清除筛选按钮
```

### Error 状态
```
网络错误：断网插画 + "网络连接失败" + 重试按钮
服务端错误：错误插画 + "服务器繁忙" + 重试按钮
权限不足："无访问权限" + 说明原因 + 联系管理员链接
404：品牌化 404 页面 + 返回首页按钮
```

### Edge 状态处理
```
超长文本：单行用 text-overflow: ellipsis；多行用 -webkit-line-clamp: 3
无空格长字符串（URL/ID）：word-break: break-all 或 overflow-wrap: break-word
大数字：千分位分隔（1,000,000）；中文用万/亿（>=10000 = "1.2万"）
日期时间：列表/表格用 YYYY-MM-DD；详情页用 YYYY年MM月DD日 HH:mm
相对时间：<1小时="X分钟前"，<24小时="X小时前"，>24小时=显示日期
极多数据：分页（>10行必须有分页）或虚拟滚动（>1000行）
```

> 表单验证完整状态机详见 → `form-validation.md`

## 操作反馈状态

```
按钮点击 → 立即变为 loading 状态（禁用 + spinner）
表单提交成功 → Toast 2-3 秒自动消失
表单提交失败 → 字段旁显示红色错误信息（说明原因和修复方式）
删除/破坏性操作 → 确认对话框（显示内容名称）
复制/分享 → "已复制" Toast
```

## 检查清单

每个列表/表格/表单页面交付前确认：
- [ ] 有骨架屏/加载动画？
- [ ] 空数据时有引导操作？
- [ ] 网络错误有重试？
- [ ] 超长文本有截断？
- [ ] 超多数据有分页？
- [ ] 表单提交有 loading + success + error 三态？

## 铁律

```
铁律: 每个数据展示组件必须覆盖五态（Loading/Empty/Error/Populated/Edge）。
铁律: Loading 状态必须用骨架屏，不是空白，不是 Spinner。
铁律: Empty 状态必须有行动引导，不能只写"暂无数据"。
```

## 红旗列表

| 你在想... | 现实 |
|-----------|------|
| "这个组件不会有空状态" | 每个数据组件都会有空状态。第一次使用时、筛选无结果时、删除全部后。 |
| "Loading 用个 Spinner 就行" | Spinner 不传达内容结构。骨架屏让用户预知布局。 |
| "Edge 状态不重要" | 用户名 50 个字符、列表 10000 条、价格为 0 — 这些都是真实场景。 |
| "错误状态交给后端处理" | 前端必须展示友好的错误信息 + 重试按钮。后端错误码不是给用户看的。 |
| "这个页面数据一定存在" | 不存在"一定存在"的数据。网络断开呢？权限不足呢？ |

## 合理化预防

| 借口 | 现实 |
|------|------|
| "时间不够做五态" | 骨架屏 + 空状态只需 10 分钟。缺它们 = 用户流失。 |
| "用户不会遇到这些情况" | 你的假设不是事实。五态是防御性设计。 |
| "先做 Populated，后面补" | "后面"永远不会来。现在做。 |
| "这个表格不会有边界情况" | 列名 30 个字符、单元格内容 200 字符、空表格 — 都是边界。 |
