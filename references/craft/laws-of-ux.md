# 认知心理学定律 — UI 设计决策依据

> 每条定律都有具体的代码生成指令，不是空洞的理论。

## 核心定律速查表

| 定律 | 核心观点 | 设计指令 |
|------|----------|----------|
| **Hick's Law** | 选项越多，决策时间越长（对数关系） | 每屏不超过 7 个选项；渐进式披露；合理默认值；不常用选项折叠到"高级设置" |
| **Miller's Law (7±2)** | 短期记忆容量 7±2 个信息块 | 信息分块；视觉层级不超过 4 层；电话号码分段显示（3-4段）；导航项 <= 7 |
| **Fitts's Law** | 到达目标的时间 = f(距离, 大小) | 主要按钮更大更近；触控目标 >= 44x44px；高频操作在拇指热区；屏幕边角是"无限大"目标 |
| **Jakob's Law** | 用户大部分时间花在别人的产品上 | 遵循约定俗成的布局（80% 成功率 vs 创新布局仅 9%）；在细节上创新，不在核心可用性上创新 |
| **Doherty Threshold (400ms)** | 响应超过 400ms 就打断注意力 | 非即时操作必须有加载反馈（骨架屏/进度条）；动画加速感知速度 |
| **Von Restorff** | 与众不同的元素更容易被记住 | CTA 按钮必须与页面其他元素视觉差异明显；一个页面只有一个"最突出"元素 |
| **Peak-End Rule** | 用户只记得最高点和结尾 | 关键成就时刻加动效庆祝；结尾体验（结账/注册完成/提交成功）必须顺畅愉快 |
| **Zeigarnik Effect** | 未完成的任务更容易被记住 | 进度条、任务清单、"你还差一步"提示——利用未完成感驱动用户继续 |
| **Serial Position Effect** | 用户最容易记住序列的首项和末项，中间项最易遗忘 | 导航栏：核心入口放首位和末位；产品卖点列表：最有吸引力的放开头和结尾；表单字段：关键项放开头或结尾；不要把转化按钮藏在长页面正中间 |
| **Aesthetic-Usability** | 好看的设计被感知为更好用 | 视觉质量直接影响用户信任度；但不能用美观掩盖功能缺陷 |
| **Gestalt: Proximity** | 靠近的元素被感知为一组 | 相关表单项/按钮/标签聚在一起；不相关的项目用间距隔开 |
| **Gestalt: Similarity** | 相似的元素被感知为同类 | 导航链接共享字体/颜色；CTA 用对比样式 |
| **Gestalt: Continuity** | 眼睛跟随平滑路径 | 对齐线和引导线引导视线；列表项左对齐形成视觉流 |
| **Gestalt: Closure** | 大脑自动补完不完整形状 | 简化图标设计；用户"参与"补全，提升记忆度 |
| **Gestalt: Figure-Ground** | 前景与背景的区分 | 模态框用半透明背景推远底层内容；浮层用阴影建立层级 |
| **Gestalt: Common Region** | 同一边界内的元素被感知为一组 | 用边框、背景色或卡片容器创建视觉分组；侧栏用不同背景色与主内容区分；功能区块用卡片包裹而非散落排列 |

## 认知负荷理论（3 种类型）

| 类型 | 定义 | 设计目标 |
|------|------|----------|
| **内在认知负荷** | 任务本身的固有难度（税表 vs 社交帖子） | 设计无法改变，接受它 |
| **外在认知负荷** | 糟糕设计带来的额外负担——找不到按钮、图标含义不明、错误信息模糊 | **主要设计目标：最小化这个** |
| **关联认知负荷** | 学习系统模式的有益投入 | 适度刺激——一致的交互模式让用户"学会"系统，越用越快 |

### 最小化外在认知负荷的具体措施

1. **图标必须有文字标签**（除非 100% 通用图标如放大镜=搜索）
2. **错误信息必须说"怎么修"而不只是"出了什么错"**
3. **表单分组用 proximity + 视觉分隔**（fieldset + legend 或卡片分组）
4. **长表单拆成多步 + 进度条**（利用 Zeigarnik Effect 驱动完成）
5. **导航深度不超过 3 层**（超过 3 层用面包屑 + 搜索补充）
6. **每屏只传达一个核心信息**（用户不需要同时看到"统计 + 列表 + 设置"）

## 说服性设计与伦理红线

### 可使用的说服原理

| 原理 | 机制 | UI 表现 |
|------|------|---------|
| **社会认同** | 人们参照他人行为 | 评价数、用户量、"XX 人正在浏览" |
| **稀缺性** | 感知稀缺提升价值 | 倒计时、"仅剩 3 件"、限时优惠 |
| **锚定效应** | 过度依赖第一个数字 | 原价划线 + 折后价（原价=锚点） |
| **损失厌恶** | 损失痛感 ≈ 同等收益快感的 2 倍 | 试用到期提醒、"你将失去 XX 功能" |

### 暗黑模式（绝对禁止）

| 类别 | 描述 | 例子 |
|------|------|------|
| **纠缠（Nagging）** | 无法关闭的弹窗、每次启动都弹评分 | 关闭按钮灰色/隐藏 |
| **阻碍（Obstruction）** | 故意让取消/退订变得极其困难 | 必须打电话才能取消订阅 |
| **偷渡（Sneaking）** | 预选额外付费项、最后一步才显示隐藏费用 | 默认勾选保险 |
| **界面干扰** | 广告伪装成系统通知 | 用通知样式推广告 |
| **强制操作** | 注册时强制分享通讯录 | 不授权就不让用 |

### 三条伦理测试

1. **用户事后会感激还是愤怒？** 如果发现被引导做了违背自身利益的决定 → 操控
2. **界面是否损害了理性决策能力？** 利用信息不对称或认知偏差是不道德的，即使合法
3. **团队能否公开为此设计辩护？** 经不起公众审视就重新考虑

## AARRR 海盗模型（设计关注点）

| 阶段 | 问题 | 设计重点 |
|------|------|----------|
| **获取 Acquisition** | 用户怎么找到你？ | 落地页转化、首屏吸引力 |
| **激活 Activation** | 是否获得首次价值体验？ | 引导流程、缩短到达"Aha Moment"的时间 |
| **留存 Retention** | 是否回来？ | 留存曲线 = PMF 验证。健康：早期急降后平台。不健康：趋向零 |
| **变现 Revenue** | 如何盈利？ | 付费墙时机——太早=用户反感，太晚=错过窗口 |
| **推荐 Referral** | 是否推荐？ | 分享机制便捷性、社交货币创造 |

## Hook 模型（Nir Eyal 四步习惯循环）

1. **触发（Trigger）**：外部（推送/邮件）+ 内部（无聊/FOMO）。目标：将产品与用户内在情绪关联
2. **行动（Action）**：触发后最简单的操作。基于 Fogg 行为模型：动机 + 路径 + 低门槛同时具备
3. **多变的酬赏（Variable Reward）**：不可预测的奖励维持兴趣。三类：社交酬赏/猎物酬赏/自我酬赏
4. **投入（Investment）**：用户投入时间/数据/社交关系 → 提升产品价值 + 提高切换成本 → 飞轮效应
