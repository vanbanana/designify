# Visual Rhythm — 视觉节奏

> 均匀的间距不是安全，是乏味。
> 一个模板化的页面，最明显的特征是每个 section 之间的距离完全一样。

---

## 核心概念

视觉节奏 = Section 间距的刻意变化。它不是随机的，而是**有意图的强弱交替**。

```
❌ 模板节奏（每个 section 间距一样）:
[Section A] ← 48px → [Section B] ← 48px → [Section C] ← 48px → [Section D]

✅ 设计节奏（间距有变化）:
[Section A] ← 64px → [Section B] ← 32px → [Section C] ← 48px → [Section D]
```

**为什么均匀间距是问题？**
- 均匀间距 = 页面没有"呼吸"——既没有紧张时刻（紧凑拉近相关性），也没有释放时刻（留白让内容沉淀）
- 从 Gestalt 心理学角度（laws-of-ux.md），间距是分组信号。全部一样的间距意味着"全部一样重要"——这是假设计，不是设计

**心理学依据：**
- **Gestalt: Proximity**（laws-of-ux.md）：靠近的元素被感知为一组。Section 间距变化就是告诉用户"这个 section 和上一个的关系远近"
- **Serial Position Effect**（laws-of-ux.md）：用户最容易记住首项和末项。页面开头和结尾的 section 应该获得更多空间，中间的 section 可以紧凑
- **Peak-End Rule**（laws-of-ux.md）：用户只记得最高点和结尾。最后一个 section 的间距决定用户的"结尾体验"

## 节奏模式

### 基础模式：密集 → 开放 → 中等

这是最安全也最有效的节奏模式：

```
┌──────────────────────────────┐
│  Hero / 开头                  │ ← 最大间距（px-16 到 px-24）
│  （给用户呼吸空间）            │
├──────────────────────────────┤
│                               │
│  内容密集区                    │ ← 中等间距（px-12 到 px-16）
│  （多卡片、网格、列表）        │
│                               │
├──────────────────────────────┤
│  关键信息 / CTA               │ ← 开放间距（px-16 到 px-20）
│  （让用户停下来看）            │
├──────────────────────────────┤
│  次要内容 / 页脚              │ ← 紧凑间距（px-8 到 px-12）
│  （用户知道这是结尾）          │
└──────────────────────────────┘
```

### 进阶模式：三节拍

```
[密集] → [开放] → [密集] → [开放]
   ↑        ↑
  信息密集    视觉呼吸
  多文字      少文字/大留白
  卡片网格    单焦点元素
```

- **密集节拍**：信息丰富，多行文字，卡片网格，表格 → 用较小间距（px-8 到 px-12）
- **开放节拍**：一个核心元素，大标题，插画，全宽图片 → 用较大间距（px-16 到 px-24）
- 交替出现，形成"呼吸节奏"

### 间距速查表

| 节奏类型 | Section 间距 | 适用场景 | 占比建议 |
|----------|-------------|----------|----------|
| **紧凑** | 24-32px（px-6 到 px-8） | 内容列表、数据密集区、关联 section | 30% |
| **中等** | 48-64px（px-12 到 px-16） | 默认节奏、内容过渡、多数 section | 40% |
| **开放** | 64-96px（px-16 到 px-24） | Hero、CTA、关键信息、结尾 | 30% |

## "间距即层级"原则

一个元素周围的**空白空间**比它的**字号**更能传达重要性。

```
❌ 用小字号+大间距（矛盾信号）:
  一个小字体元素周围有 96px 空白 → 用户困惑它是重要还是不重要的

✅ 间距与视觉权重一致:
  大标题 + 48px 上方间距 + 24px 下方间距 = "这是本页最重要的东西"
  辅助文字 + 16px 上方间距 + 8px 下方间距 = "这是次要说明"
```

### 标题间距规则

一个标题属于它下方的**内容**，不属于它上方的空白：

```
✅ 正确:
  ┌───────────────────────┐
  │  (48px 上方间距)       │ ← 标题属于下方，与上方疏远
  │  标题                   │
  │  (16px 下方间距)       │ ← 标题与下方内容紧密
  │  内容段落...            │
  └───────────────────────┘
  
❌ 错误（标题居中，不知道属于哪个）:
  │  标题                   │
  │  (32px)                │ ← 上下间距一样
  │  内容段落...            │
```

## 布局不对称

**对称布局 = 安全 = 没人注意。不对称布局 = 一个 bold have。**

但不对称不是随便偏移，必须基于内容权重：

```
✅ 有意的不对称（内容决定权重）:
  左侧 2/3: 主内容区（产品核心功能）
  右侧 1/3: 次要信息（相关链接）
  
✅ 故意的视觉偏移:
  标题左对齐，但右侧有一个偏移的视觉元素（色块/图片）平衡
  
❌ 无意义的不对称:
  只是把卡片往右移了 10px 然后说"这是不对称设计"
```

### 不对称布局的心理学依据

- **Von Restorff 效应**：不对称本身就是"与众不同的元素"，用户会记住它
- **Gestalt: Closure**：大脑会自动补全不完整的形状。不对称布局让用户"参与"补全，提升参与感
- **但**：Jakob's Law 告诉我们——核心操作必须遵循约定。不对称用在装饰性区域，不在导航/CTA 等核心交互上

## 反模式

| 反模式 | 为什么是问题 | 正确做法 |
|--------|-------------|----------|
| 所有 section 间距完全相等 | 模板标志，没有呼吸感 | 交替使用 px-8/px-12/px-16 |
| 卡片内部间距 > 卡片之间间距 | 违反亲密性原则——卡片之间距离太大，看起来不相关 | 卡片间距 ≤ 卡片内边距 × 2 |
| 标题上间距 < 标题下间距 | 标题看起来属于上方的 section | 标题上间距 ≥ 标题下间距 × 2 |
| 手机端用和桌面端完全相同的节奏 | 手机屏幕小，需要更紧凑的节奏 | 手机端所有间距减半，但仍保持变化 |
| 每个 section 内部间距一样 | section 内部也需要节奏 | 用"标题+内容"的微节奏变化 |

---

## 铁律

```
铁律: 一页之内所有 Section 间距不能完全相同。至少有两种间距值。（质量门禁检查项）
铁律: 标题下方的间距 < 标题上方的间距。（标题属于下方内容）
铁律: Section 间距必须用 4 的倍数，且至少使用 2 个不同值。
```

## 红旗列表

| 你在想... | 现实 |
|-----------|------|
| "所有间距用 48px 最统一" | 统一 = 乏味。48px 本身没问题，但只有 48px = 模板。 |
| "用户不会注意到间距变化" | 用户确实"注意不到"——但他们能感受到。无意识层面的"这个页面感觉对"或有"这个页面很普通"。 |
| "间距变化不一致的话设计看起来乱" | 不基于内容的随机变化才乱。基于内容权重（密集→开放→中等）的节奏是设计。 |
| "先做均匀的，后面再调" | 后面永远不会调。现在做。 |
