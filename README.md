Every AI coding assistant generates frontend code that **looks** professional but **fails** at:

| ❌ AI Slop | Why It Happens |
|------------|---------------|
| Purple-blue gradients everywhere | Training data bias — #6366f1 is the most common color in GitHub repos |
| 3-column equal-width cards | Most common layout pattern in training data |
| Lorem ipsum / "John Doe" | No mechanism to force real content |
| No empty/loading/error states | AI only generates the "happy path" |
| All buttons look identical | No hierarchy understanding |
| No accessibility | No WCAG enforcement |
| Spacing feels "off" | No 4px grid system enforcement |

**Designify solves all of these** with a 7-phase workflow, 13 craft rule files, 74 real brand design systems, and a zero-dependency website reverse-engineering tool.

## 🧠 Psychology-Driven Design

Every design decision has a cognitive science basis. No "I think this looks good."

| UX Law | Design Rule | Enforcement |
|--------|------------|-------------|
| **Hick's Law** | ≤ 7 options per screen; progressive disclosure | Checked in Phase 6 |
| **Fitts's Law** | Touch targets ≥ 44×44px; primary buttons larger | Quantitative gate |
| **Miller's Law** | ≤ 4 visual hierarchy levels | Design Read required |
| **Jakob's Law** | Follow conventions (80% success vs 9% for novel) | Brand routing |
| **Serial Position** | Key items at start and end of lists | Layout decision tree |
| **Von Restorff** | One "most prominent" element per page | Anti-slop P0 check |
| **Zeigarnik Effect** | Progress bars drive task completion | State coverage |
| **Gestalt: Proximity** | Related items clustered; unrelated spaced apart | Spacing rules |

## 🎯 7-Phase Workflow

### Phase 1: Discovery (Questioning Protocol)
3 rounds of structured questions. Even zero-experience users become product architects.
- **Q1**: Product type (multiple choice)
- **Q2**: Target user + problem (guided reasoning)
- **Q3**: Style preference (natural language → 8-category mapping)
- **Bonus**: Reference URL? → Run `website-analyzer.mjs` for pixel-level extraction

### Phase 2: Product Definition
Full architecture document with AARRR user flows. Written to `output/YYYY-MM-DD-product-definition.md`. **User must approve before proceeding.**

### Phase 3: Design Read ⛔ HARD GATE
**No code without a Design Read.** This is the Iron Law. Every project — no matter how "simple" — must output:
- Project type → route to category file
- Style inference → route to brand DESIGN.md
- Psychology basis → which UX laws apply and why
- Design parameters → density, motion, visual richness
- Material plan → icons, illustrations, images, textures

### Phase 4: Design Direction
- Layout decision tree (F-pattern / Z-pattern / Grid / Sidebar / etc.)
- 8-category style routing to 74 real brand design systems
- Hard design rules (spacing 4px multiples, contrast 4.5:1, touch 44px, etc.)

### Phase 5: Code Generation
7-step sequential generation: Skeleton → Tokens → Assets → Content → Interaction → States → Motion

### Phase 6: Quality Gate + Critique Theater
10-dimension quantitative checklist. Anti-AI-Slop P0 check (6 instant-fail items). Up to 3 rounds of self-review.

### Phase 7: Delivery + Progress Ledger
Quantified results with evidence. Progress persisted to `output/progress.md` for context-compaction recovery.

## 🛡️ Anti-AI-Slop System

### P0 — Instant Fail (must fix before delivery)
| Slop Pattern | What To Do Instead |
|-------------|-------------------|
| Purple-blue gradient `#6366f1 → #8b5cf6` | Use brand-routed primary color |
| Tailwind indigo/blue as primary | Route through `brands/[brand].md` |
| Decorative glass morphism | Only use `backdrop-filter` with purpose |
| 3-column equal-width cards | Use weighted grid with hierarchy |
| Lorem ipsum / "John Doe" | Real business content, always |
| Only default state, no empty/loading/error | 5-state coverage mandatory |

### Enforcement Architecture
```
Iron Laws → Red Flags → Rationalization Tables → Evidence-Before-Claims
     │            │              │                        │
     ▼            ▼              ▼                        ▼
 "NO CODE    "STOP when     "Excuse → Reality"     "Show data,
  WITHOUT      thinking                           not opinions"
  DESIGN       these..."
  READ"
```

Inspired by [obra/superpowers](https://github.com/obra/superpowers) — the gold standard in agent behavior shaping.

## 🚀 Quick Start

### 1. Install as a Skill

**Claude Code:**
```bash
# Clone into your skills directory
cp -r designify/ ~/.claude/skills/designify/
```

**Cursor:**
```bash
# Copy to your .cursor/rules directory
cp -r designify/ .cursor/rules/designify/
```

**QoderWork / Other agents:**
```bash
# Place in your agent's skills directory
cp -r designify/ <your-skills-dir>/designify/
```

### 2. Use It

Just tell your AI agent:
```
帮我做一个 SaaS 后台
```

Or with a reference:
```
帮我复刻 https://linear.app 的风格做一个项目管理工具
```

The agent will automatically:
1. Ask structured questions (even if you're a beginner)
2. Route to the right project type + brand design system
3. Apply psychology-driven design decisions
4. Generate code with quantitative quality gates
5. Self-review with Critique Theater
6. Deliver with evidence

### 3. Analyze Any Website

```bash
# Zero-dependency website reverse engineering
node tools/website-analyzer.mjs https://stripe.com --output analysis.md

# With screenshot
node tools/website-analyzer.mjs https://linear.app --output analysis.md --screenshot screenshot.png
```
